# HITL feedback round 2

Tue, 21 Apr 26

### Team Assist UI/UX Design Direction

- Consolidate AI interactions into conversational interface
  - Move away from separate task cards to unified chat experience
  - AI requests should appear in same UI as human-AI handoffs
  - Timeline shows only actual customer messages
  - Team Assist panel handles all AI collaboration elements
- Conversational approval flow
  - AI asks: “Hey, I need help with return exception. Here’s customer background. Should I do this?”
  - Human responds naturally vs commenting on tasks
  - Responses persist as objects in timeline for history

### Pricing Strategy Discussion

- Early access pricing: per agent per month for Team Assist
  - Volume-based model like Claude ($20/month with usage limits)
  - Credit system without billing mechanism initially
  - Bundle allocation per agent with overage options
- Implementation approach
  - Start with generous limits to avoid friction
  - Monitor weekly usage data to refine pricing
  - Structure allows future credit card billing for CX leaders
  - Sales team concerns about complexity vs future-proofing needs

### Demo Requirements & Messaging

- Approval scenarios for prototype
  - Show “yes/no” responses vs formal approval language
  - Include examples of human modifications (AI suggests $25 credit, human counters with $10)
  - Demonstrate positive interactions (increasing credits vs denying)
- Audience considerations
  - May need to reference familiar AI tools (Claude, code assistants) as setup
  - Avoid technical/database language in UI
  - Use plain English (e.g., “VIP customer” vs “customer_tier_VIP”)

### Technical Implementation Status

- Greg’s team building in environment today
- Christina designing interface iterations
- Early Team Assist prototypes from last year provide foundation
- Timeline: Demo ready today or tomorrow

### Demo Logistics

- Duration: 2-5 minutes for maximum impact
- Food ordered (tacos processed)
- Matt briefed on current direction vs original space concept

### Navigation and AI Integration Decisions

- Middle nav design approved for GCL launch
  - Will show on left side navigation
  - Conversational analytics button added below reports
  - Full screen view for analytics experience
- AI co-pilot panel integration
  - Universal AI button across admin pages where AI can help
  - Panel opens on right side, consistent styling
  - Multiple entry points based on page context
- Team assist AI engagement
  - Within guides: Steve to decide implementation approach
  - No add button for GCL initially
  - Future: persistent admin AI agent across Gladly Connect

### Human-in-the-Loop (HITL) Product Design

- Core experience flows through team assist panel
  - Customer list shows indicators for AI decisions vs conversations
  - Timeline displays messages, AI decisions happen in right panel
  - Agents work primarily within panel interface
- Decision approval workflow
  1. AI proactively communicates status (“I’ll be right with you”)
  2. Presents recommendation requiring approval
  3. Pre-populated response options (approve/deny/modify)
  4. Sends response with indicator of AI assistance
- Integration challenges
  - External system workflows need accommodation
  - Agent reports back completion for AI summary
  - Moving toward simpler integration experience with new team platform

### Pricing and Packaging Strategy

- HITL packaged within team assist pricing
  - Usage-based model preferred over per-agent
  - Avoids double-charging customers
  - Resolution vs assist pricing needs reconciliation
- Early access approach
  - Free usage for first 3 months
  - Set high usage expectations upfront
  - Gather data before final pricing model
- Cost tracking concerns
  - LLM costs for simulations: ~$15 for 545 simulations
  - Evals pricing depends on scheduling and conversation volume
  - Team assist usage tracking not implemented yet
